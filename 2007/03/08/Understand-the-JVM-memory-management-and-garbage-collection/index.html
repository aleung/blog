дәҶи§ЈJVMзҡ„еҶ…еӯҳз®ЎзҗҶдёҺеһғеңҫеӣһж”¶

еҸ‘иЎЁдәҺ 2007-03-08 жӣҙж–°дәҺ 2016-03-16

JavaиҜӯиЁҖе…·еӨҮGC(еһғеңҫеӣһж”¶)зҡ„иғҪеҠӣпјҢеҶ…еӯҳз®ЎзҗҶдёҚйңҖиҰҒеә”з”ЁзЁӢеәҸеҺ»иҝҮй—®пјҢиҝҷеҫҲж–№дҫҝгҖӮдҪҶжҳҜпјҢGCжҳҜжҖҺд№ҲиҝӣиЎҢзҡ„пјҢJVMзҡ„еҶ…еӯҳеҸӮж•°еә”иҜҘжҖҺд№Ҳи°ғж•ҙпјҢеҰӮдҪ•дјҳеҢ–пјҢеҫҖеҫҖжҲ‘д»¬дёҚжҳҜеӨӘжё…жҘҡгҖӮзңӢиҝҮдёҖдәӣиө„ж–ҷеҗҺпјҢеҜ№Sun JVMзҡ„еҶ…еӯҳз®ЎзҗҶд»ҘеҸҠеһғеңҫеӣһж”¶зҡ„жңәеҲ¶еӨ§жҰӮжңүдәҶдёҖдёӘжҰӮеҝөпјҢиҝҷйҮҢе°Ҷиҝҷдәӣиө„ж–ҷеҪ’зәіе’Ңзҝ»иҜ‘еҮәжқҘгҖӮжң¬ж–ҮеҶ…е®№дё»иҰҒеҹәдәҺSun JVM 1.3.1пјҢеңЁеҗҺз»ӯзүҲжң¬дёӯжңүдёҚе°‘дјҳеҢ–жҺӘж–ҪпјҢдҪҶжҳҜиҝҷдәӣеҹәжң¬жҰӮеҝөиҝҳжҳҜдёҚеҸҳзҡ„гҖӮ

иҝҷйҮҢеҒҮи®ҫеӨ§е®¶еҜ№GCзҡ„жҰӮеҝөе’Ңеҹәжң¬еҺҹзҗҶйғҪе·Із»ҸдәҶи§ЈпјҢдёҚиҜҰз»ҶеҸҷиҝ°дәҶгҖӮ

еҪ“JVMиҝӣиЎҢGCзҡ„ж—¶еҖҷпјҢжҳҜиҰҒж¶ҲиҖ—CPUиө„жәҗе’ҢйңҖиҰҒдёҖе®ҡж—¶й—ҙзҡ„пјҢиҝҷдјҡеҪұе“ҚеҲ°зЁӢеәҸзҡ„жӯЈеёёиҝҗиЎҢпјҢеӣ жӯӨйңҖиҰҒе°ҪеҸҜиғҪеҮҸе°‘GCж¶ҲиҖ—зҡ„ж—¶й—ҙгҖӮJavaзЁӢеәҸиҝҗиЎҢиҝҮзЁӢдёӯпјҢеҜ№иұЎзҡ„з”ҹе‘Ҫе‘Ёжңҹжңүй•ҝжңүзҹӯпјҢе…¶дёӯзӣёеҪ“еӨ§йғЁеҲҶжҳҜйғҪжҳҜжҜ”иҫғзҹӯе‘Ҫзҡ„пјҢдҫӢеҰӮеұҖйғЁзҡ„еҜ№иұЎдёҖз”Ёе®Ңе°ұеҸҜд»Ҙеӣһж”¶дәҶгҖӮеңЁеӨ§еӨҡж•°жғ…еҶөдёӢпјҢеҸӘиҰҒиғҪеӨҹеҸҠж—¶еӣһж”¶иҝҷдәӣзҹӯе‘ҪеҜ№иұЎзҡ„еҶ…еӯҳпјҢе°ұиғҪеӨҹзЎ®дҝқJVMжңүи¶іеӨҹеҶ…еӯҳжқҘеҲҶй…Қз»ҷж–°зҡ„еҜ№иұЎгҖӮеӣ жӯӨJVMйҮҮз”ЁдёҖз§ҚеҲҶд»Јеӣһж”¶(generational collection) зҡ„зӯ–з•ҘпјҢз”Ёиҫғй«ҳзҡ„йў‘зҺҮеҜ№е№ҙиҪ»зҡ„еҜ№иұЎ(young generation)иҝӣиЎҢжү«жҸҸе’Ңеӣһж”¶пјҢиҝҷз§ҚеҸ«еҒҡminor collectionпјҢиҖҢеҜ№иҖҒеҜ№иұЎ(old generation)зҡ„жЈҖжҹҘеӣһж”¶йў‘зҺҮиҰҒдҪҺеҫҲеӨҡпјҢз§°дёәmajor collectionгҖӮиҝҷж ·е°ұдёҚйңҖиҰҒжҜҸж¬ЎGCйғҪе°ҶеҶ…еӯҳдёӯжүҖжңүеҜ№иұЎйғҪжЈҖжҹҘдёҖйҒҚгҖӮ

Sun JVM 1.3 жңүдёӨз§ҚжңҖеҹәжң¬зҡ„еҶ…еӯҳж”¶йӣҶж–№ејҸпјҡдёҖз§Қз§°дёәcopyingжҲ–scavengeпјҢе°ҶжүҖжңүд»Қз„¶з”ҹеӯҳзҡ„еҜ№иұЎжҗ¬еҲ°еҸҰеӨ–дёҖеқ—еҶ…еӯҳеҗҺпјҢж•ҙеқ—еҶ…еӯҳе°ұеҸҜеӣһж”¶гҖӮиҝҷз§Қж–№жі•жңүж•ҲзҺҮпјҢдҪҶйңҖиҰҒжңүдёҖе®ҡзҡ„з©әй—ІеҶ…еӯҳпјҢжӢ·иҙқд№ҹжңүејҖй”ҖгҖӮиҝҷз§Қж–№жі•з”ЁдәҺminor collectionгҖӮеҸҰеӨ–дёҖз§Қз§°дёәmark-compactпјҢе°Ҷжҙ»зқҖзҡ„еҜ№иұЎж Үи®°еҮәжқҘпјҢз„¶еҗҺжҗ¬иҝҒеҲ°дёҖиө·иҝһжҲҗеӨ§еқ—зҡ„еҶ…еӯҳпјҢе…¶д»–еҶ…еӯҳе°ұеҸҜд»Ҙеӣһж”¶дәҶгҖӮиҝҷз§Қж–№жі•дёҚйңҖиҰҒеҚ з”ЁйўқеӨ–зҡ„з©әй—ҙпјҢдҪҶйҖҹеәҰзӣёеҜ№ж…ўдёҖдәӣгҖӮиҝҷз§Қж–№жі•з”ЁдәҺmajor collection.

еңЁJVM 1.3еҸҠд»ҘеҗҺзҡ„зүҲжң¬дёӯпјҢиҝҳжңүе…¶д»–еҸҜйҖүзҡ„еҶ…еӯҳж”¶йӣҶж–№жі•пјҢйҖҡиҝҮзү№е®ҡзҡ„еҸӮж•°жқҘи®ҫе®ҡгҖӮдҫӢеҰӮпјҡеўһйҮҸејҸеӣһж”¶пјҢжҜҸж¬ЎеҸӘеӨ„зҗҶдёҖе°ҸйғЁеҲҶпјӣжӣҝд»ЈеҚ•зәҝзЁӢcopyingзҡ„еӨҡзәҝзЁӢе№¶иЎҢеӣһж”¶пјӣжӣҝд»Јmark-compactзҡ„concurrent mark-sweepеӣһж”¶зӯүзӯүгҖӮеҸӮиҖғиө„ж–ҷ[4][5]дёӯжңүжӣҙеӨҡжҸҸиҝ°гҖӮ

JVMз®ЎзҗҶзҡ„еҶ…еӯҳпјҢйҖҡеёёеҸ«еҒҡе Ҷ(heap)пјҢеҸҜд»Ҙз”ЁдёӢйқўзҡ„еӣҫжқҘжҸҸиҝ°гҖӮ

JVMеҗҜеҠЁеҗҺпјҢдҝқз•ҷдёҖж®өең°еқҖз©әй—ҙпјҢиҝҷдёӘз©әй—ҙзҡ„еӨ§е°Ҹз”ұ-XmxжҢҮе®ҡгҖӮиҝҷеқ—з©әй—ҙзҡ„еӨ§е°Ҹе°ұжҳҜheapеҸҜиғҪзҡ„жңҖеӨ§еҖјпјҢдҪҶдёҖејҖе§ӢдёҚдёҖе®ҡе…ЁйғҪеҲҶй…ҚдәҶзү©зҗҶеҶ…еӯҳпјҢеҲқе§ӢеҲҶй…Қзҡ„heapеӨ§е°Ҹз”ұ-XmsжҢҮе®ҡпјҢеҰӮжһң-Xmsе°ҸдәҺ-XmxпјҢеү©дҪҷйғЁеҲҶжҳҜvirtualзҡ„пјҢеҪ“йңҖиҰҒзҡ„ж—¶еҖҷпјҢеҶҚеҗ‘OSз”іиҜ·гҖӮ

Young generation еҶ…еӯҳз”ұдёҖеқ—Eden(дјҠз”ёеӣӯпјҢжңүж„ҸжҖқ)е’ҢдёӨеқ—Survivor Space(1.4ж–ҮжЎЈдёӯз§°дёәsemi-space)жһ„жҲҗгҖӮж–°еҲӣе»әзҡ„еҜ№иұЎзҡ„еҶ…еӯҳйғҪеҲҶй…ҚиҮӘedenгҖӮдёӨеқ—Survivor SpaceжҖ»жңүдјҡдёҖеқ—жҳҜз©әй—Ізҡ„пјҢз”ЁдҪңcopying collectionзҡ„зӣ®ж Үз©әй—ҙгҖӮMinor collectionзҡ„иҝҮзЁӢе°ұжҳҜе°Ҷedenе’ҢеңЁз”Ёsurvivor spaceдёӯзҡ„жҙ»еҜ№иұЎcopyеҲ°з©әй—Іsurvivor spaceдёӯгҖӮжүҖи°“survivorпјҢд№ҹе°ұжҳҜеӨ§йғЁеҲҶеҜ№иұЎеңЁдјҠз”ёеӣӯеҮәз”ҹеҗҺпјҢж №жң¬жҙ»дёҚиҝҮдёҖж¬ЎGCгҖӮеҜ№иұЎеңЁyoung generationйҮҢз»ҸеҺҶдәҶдёҖе®ҡж¬Ўж•°зҡ„minor collectionеҗҺпјҢе№ҙзәӘеӨ§дәҶпјҢе°ұдјҡиў«з§»еҲ°old generationдёӯпјҢз§°дёәtenuringгҖӮ(жҳҜеҗҰд»…еҪ“survivor spaceдёҚи¶ізҡ„ж—¶еҖҷжүҚдјҡе°ҶиҖҒеҜ№иұЎtenuring? зӣ®еүҚиө„ж–ҷдёӯжІЎжңүжүҫеҲ°жҸҸиҝ°)

Permanent generation жҳҜJVMз”ЁжқҘдҝқеӯҳclass objectе’Ңmeta dataпјҢеӨ§е°Ҹз”ұ-XX:PermSizeе’Ң-XX:MaxPermSizeжҢҮе®ҡгҖӮеӨ§йҮҸеҠЁжҖҒз”ҹжҲҗ(зј–иҜ‘)е’ҢеҠ иҪҪclassдјҡеўһеҠ иҝҷйғЁеҲҶеҶ…еӯҳзҡ„иҖ—з”ЁгҖӮ

еү©дҪҷеҶ…еӯҳз©әй—ҙдёҚи¶ідјҡи§ҰеҸ‘GCпјҢеҰӮedenз©әй—ҙдёҚеӨҹдәҶе°ұиҰҒиҝӣиЎҢminor collectionпјҢold generationз©әй—ҙдёҚеӨҹиҰҒиҝӣиЎҢmajor collectionпјҢpermanent generationз©әй—ҙдёҚи¶ідјҡеј•еҸ‘full GCгҖӮ

еҫҲеӨҡеҸӮж•°дјҡеҪұе“ҚйҮҢйқўеҗ„йғЁеҲҶз©әй—ҙзҡ„еҲҶй…ҚгҖӮ-XX:MinHeapFreeRatioдёҺ-XX:MaxHeapFreeRatioи®ҫе®ҡз©әй—ІеҶ…еӯҳеҚ жҖ»еҶ…еӯҳзҡ„жҜ”дҫӢиҢғеӣҙпјҢиҝҷдёӨдёӘеҸӮж•°дјҡеҪұе“ҚGCзҡ„йў‘зҺҮе’ҢеҚ•ж¬ЎGCзҡ„иҖ—ж—¶гҖӮ-XX:NewRatioеҶіе®ҡyoungдёҺold generationзҡ„жҜ”дҫӢгҖӮYoung generationз©әй—ҙи¶ҠеӨ§пјҢminor collectionйў‘зҺҮи¶ҠдҪҺпјҢдҪҶжҳҜold generationз©әй—ҙе°ҸдәҶпјҢеҸҲеҸҜиғҪеҜјиҮҙmajor collectionйў‘зҺҮеўһеҠ гҖӮ-XX:NewSizeе’Ң-XX:MaxNewSizeзӣҙжҺҘжҢҮе®ҡдәҶyoung generationзҡ„зјәзңҒеӨ§е°Ҹе’ҢжңҖеӨ§еӨ§е°ҸгҖӮ

-Xmx
set maximum Java heap size

-Xms
set initial Java heap size

-XX:MinHeapFreeRatio=40
Minimum percentage of heap free after GC to avoid expansion.

-XX:MaxHeapFreeRatio=70
Maximum percentage of heap free after GC to avoid shrinking.

-XX:NewRatio=2
Ratio of new/old generation sizes. [Sparc -client:8; x86 -server:8; x86 -client:12.]-client:8 (1.3.1+), x86:12]

-XX:NewSize=2.125m
Default size of new generation (in bytes) [5.0 and newer: 64 bit VMs are scaled 30% larger; x86:1m; x86, 5.0 and older: 640k]

-XX:MaxNewSize=
Maximum size of new generation (in bytes). Since 1.4, MaxNewSize is computed as a function of NewRatio.

-XX:SurvivorRatio=25
Ratio of eden/survivor space size [Solaris amd64: 6; Sparc in 1.3.1: 25; other Solaris platforms in 5.0 and earlier: 32]

-XX:PermSize=
Initial size of permanent generation

-XX:MaxPermSize=64m
Size of the Permanent Generation. [5.0 and newer: 64 bit VMs are scaled 30% larger; 1.4 amd64: 96m; 1.3.1 -client: 32m.]

(дёҠйқўз»ҷеҮәзҡ„зјәзңҒеҖјдёҚдёҖе®ҡеҮҶзЎ®пјҢдёҚеҗҢJVMзүҲжң¬е’ҢдёҚеҗҢOSзҺҜеўғдёӢдјҡжңүдёҚеҗҢ)

иҝҷйҮҢз»ҷеҮәзҡ„еҸӘжҳҜеҹәжң¬зҡ„д»Ӣз»ҚпјҢдёӢйқўreferenceдёӯзҡ„ж–Үз« йғҪеҫҲдёҚй”ҷпјҢеҜ№иҝӣдёҖжӯҘдәҶи§ЈжҲ–иҖ…жҹҘжүҫжҖ§иғҪдјҳеҢ–еҸӮж•°йғҪжңүеё®еҠ©гҖӮ
